自制Xcode插件alcatraz-packages上传过程

  • ios
软件探索录

Xcode的alcatraz插件是管理Xcode插件的利器，它的安装和使用方法在网上的资料很多，这篇文章主要介绍一下自己编写的Xcode插件如何加入到alcatraz的大家庭中，供其他开发者使用。

假设你的Xcode Plugin已经编写完成，作为一个无私的分享者，你觉得自己写的东西对其他开发者有用，那就行动起来吧，按照下面的步骤：

  1. 将自己的插件工程上传到github，记下该项目地址;
  2. 找到github中alcatraz-packages的项目页面去fork（项目页面右上角按钮）到自己的账户中（当然，前提是您已经登录了github），并将fork后的项目同步到本地；
  3. 编辑本地项目根目录中的packages.json文件，alcatraz-packages的readme文件写的很清楚，不过仍然要强调的是该文件包含plugins(插件)、file_templates(文件模板)、color_schemes(主题)和project_templates(工程模板)，确保插件信息填写到plugins的大括号里；
  4. 在packages.json插件填写name,url,description,screenshot，这里注意name要与插件的工程名必须一样，记住是必须（不然，在package manager里无法安装）;
  5. 在本地的alcatraz-packages目录下，用rspec命令校验一下package.json文件，无错后同步到github，并在github的该项目页面pull request，很快便会收到提交回复。

提交通过后，在alcatraz-package的主项目页面的package.json文件会查到你编辑的内容。

其他注意事项，如果上面三步没什么差错，在Xcode的package manager里
就应该出现自己的插件了，点击install可以安装，但是到插件目录下面发现插件大小为0，或者其他问题，有个方法，可以将该插件的源码同步到本地，然后用Xcode试着编译，编译出错也是有可能的，因为git客户端在同步工程文件时有问题，笔者在用Xcode的source control时就遇到这样的bug。

如遇到这样的问题可以该插件源码目录下的.git目录删掉，重新上传，该目录为隐藏目录，mac系统默认不显示隐藏文件，可用如下命令让其现形：
defaults write com.apple.finder AppleShowAllFiles -bool true

总之，确保两点，一是所编辑、上传的packages.json符合要求，二是自己插件的源码下载后可以顺利编译。
